# ATIVIDADE 6 — O shell como JCL moderno

# O slide da genealogia mostrou que o JCL usava comandos como
# // ASSIGN SYS004,TAPE para associar um job a um dispositivo.
# No shell moderno, qual mecanismo cumpre papel equivalente?

Resposta:
O mecanismo equivalente é o Redirecionamento de Entrada/Saída (I/O Redirection) e os Pipes. No shell moderno, utilizamos símbolos como >, <, >> e | para associar o fluxo de dados de um programa a arquivos ou dispositivos, assim como o JCL fazia explicitamente.

# ATIVIDADE 7 — Make

# 1. O monitor histórico verificava a conclusão de cada job antes
#    de despachar o próximo. No make, qual mecanismo cumpre esse
#    papel? Como o make sabe que não precisa recompilar um arquivo
#    que não mudou?

Resposta 1:
O mecanismo que cumpre esse papel é a Verificação de Dependências (baseada no Grafo de Dependências). O make sabe que não precisa recompilar um arquivo comparando as marcas de tempo (timestamps): se o arquivo de destino (ex: soma.o) for mais recente que o arquivo de origem (ex: soma.c), ele entende que nada mudou e pula a tarefa.


# 2. O monitor unificava "supervisor, carregador e rotinas de I/O
#    sob uma interface textual". O que o Makefile unifica no fluxo
#    de desenvolvimento moderno?

Resposta 2:
O Makefile unifica as regras de compilação, as dependências entre arquivos e os comandos de automação (como as quatro fases do GCC que você viu) em um único arquivo de configuração. Ele abstrai o pipeline de desenvolvimento, permitindo que todo o processo de construção (build) de um software complexo seja executado com um único comando.
